# ЧАСТЬ 1: ЭКСПОРТ ПРОЕКТА ИЗ FIGMA MAKE

## Шаг 1: Скачать проект

### 1.1 В Figma Make:
1. Найдите кнопку **"Download"** или **"Export"** (обычно в правом верхнем углу)
2. Нажмите на неё
3. Выберите **"Download as ZIP"** или **"Export project"**
4. Дождитесь окончания скачивания

**📍 Где найти:** Обычно файл скачается в папку `Загрузки` (Downloads)
**📄 Название:** Что-то вроде `crm-manera.zip` или `figma-make-export.zip`

### 1.2 Распаковать архив:

**Windows:**
1. Найдите файл `.zip` в папке Загрузки
2. Правой кнопкой мыши → **"Извлечь всё..."**
3. Выберите папку, куда распаковать (например, `C:\Projects\crm-manera`)
4. Нажмите **"Извлечь"**

**Mac:**
1. Найдите файл `.zip` в папке Downloads
2. Двойной клик по файлу
3. Папка распакуется автоматически
4. Переместите её в удобное место (например, `~/Projects/crm-manera`)

**Linux:**
```bash
cd ~/Downloads
unzip crm-manera.zip -d ~/Projects/crm-manera
cd ~/Projects/crm-manera
```

✅ **Готово!** Теперь у вас есть папка с проектом.

# ЧАСТЬ 2: УСТАНОВКА НЕОБХОДИМЫХ ПРОГРАММ

## Шаг 2: Установить Node.js

### Что это?
Node.js - это программа для запуска JavaScript на компьютере. Без неё проект не запустится.

### 2.1 Скачать Node.js:

1. Откройте браузер
2. Перейдите на **https://nodejs.org**
3. Увидите 2 кнопки - выберите **LTS** (рекомендуемая версия)
4. Скачайте установщик для вашей системы

### 2.2 Установить:

**Windows:**
1. Запустите скачанный файл `node-vXX.XX.X-x64.msi`
2. Нажимайте **"Next"** → **"Next"** → **"Install"**
3. Дождитесь окончания установки
4. Нажмите **"Finish"**

**Mac:**
1. Запустите скачанный файл `node-vXX.XX.X.pkg`
2. Следуйте инструкциям установщика
3. Нажмите **"Continue"** → **"Install"**

**Linux:**
```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

# Fedora/CentOS
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### 2.3 Проверить установку:

Откройте терминал (командную строку):

**Windows:** 
- Нажмите `Win + R`
- Введите `cmd`
- Нажмите Enter

**Mac:** 
- Нажмите `Cmd + Space`
- Введите `Terminal`
- Нажмите Enter

**Проверьте:**
```bash
node --version
```

Должно показать что-то вроде: `v20.10.0`

```bash
npm --version
```

Должно показать что-то вроде: `10.2.3`

✅ **Если видите версии - всё установлено правильно!**

❌ **Если ошибка** - переустановите Node.js и перезагрузите компьютер.

# ЧАСТЬ 3: НАСТРОЙКА БАЗЫ ДАННЫХ (SUPABASE)

## Шаг 4: Создать проект в Supabase

### Что это?
Supabase - это бесплатная облачная база данных. Здесь будут храниться все данные вашего CRM.

### 4.1 Регистрация:

1. Перейдите на **https://supabase.com**
2. Нажмите **"Start your project"**
3. Войдите через:
   - GitHub (рекомендуется)
   - Или Email

### 4.2 Создать проект:

1. После входа нажмите **"New Project"**
2. Заполните форму:

   **Organization:** 
   - Если первый раз - создайте организацию (введите любое название, например "Manera")
   
   **Name:** `crm-manera`
   
   **Database Password:** 
   - Придумайте сложный пароль (или используйте сгенерированный)
   - ⚠️ **ВАЖНО:** Сохраните этот пароль! Запишите в блокнот!
   
   **Region:** 
   - Выберите ближайший регион (для России - `Europe (Frankfurt)` или `Central EU (Warsaw)`)
   
   **Pricing Plan:** 
   - Выберите **Free** (бесплатно)

3. Нажмите **"Create new project"**
4. ⏳ Подождите 2-3 минуты (Supabase создаёт вашу базу данных)

✅ **Проект создан!** Переходим к следующему шагу.

---

## Шаг 5: Запустить SQL миграции

### Что это?
SQL миграции - это скрипты, которые создают таблицы в базе данных.

### 5.1 Открыть SQL Editor:

1. В Supabase Dashboard (где вы сейчас находитесь)
2. В левом меню найдите и нажмите **"SQL Editor"** (иконка с символом `</>`)

### 5.2 Запустить первую миграцию:

1. Нажмите **"+ New query"** (вверху слева)
2. В вашем проекте откройте файл `/supabase/migrations/001_initial_schema.sql`
3. **Скопируйте ВСЁ** содержимое файла (Ctrl+A, Ctrl+C)
4. Вернитесь в Supabase SQL Editor
5. **Вставьте** скопированный текст в редактор (Ctrl+V)
6. Нажмите **"Run"** (или `Ctrl + Enter`)
7. ⏳ Подождите 5-10 секунд
8. ✅ Должно появиться **"Success. No rows returned"**

### 5.3 Запустить вторую миграцию:

1. Очистите редактор (выделите всё и удалите)
2. Откройте файл `/supabase/migrations/002_rls_policies.sql`
3. Скопируйте ВСЁ содержимое
4. Вставьте в Supabase SQL Editor
5. Нажмите **"Run"**
6. ✅ Дождитесь успешного выполнения

### 5.4 Запустить третью миграцию (тестовые данные):

1. Очистите редактор
2. Откройте файл `/supabase/migrations/003_seed_data.sql`
3. Скопируйте ВСЁ содержимое
4. Вставьте в Supabase SQL Editor
5. Нажмите **"Run"**
6. ✅ Готово!

### 5.5 Проверить что таблицы созданы:

1. В левом меню нажмите **"Table Editor"**
2. Вы должны увидеть список таблиц:
   - `users`
   - `students`
   - `parents`
   - `teachers`
   - `groups`
   - И другие...

✅ **Если видите таблицы - база данных готова!**

Вы увидите несколько полей. Нам нужны два:

**1. Project URL:**
```
https://abcdefghijklmnop.supabase.co
```
- Нажмите на иконку 📋 **"Copy"** справа

**2. API Key (anon/public):**
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```
- Найдите ключ с подписью `anon` `public`
- Нажмите **"Reveal"** (показать)

⚠️ **ВАЖНО:** Сохраните оба ключа в блокноте! Они нам понадобятся через минуту.

# ЧАСТЬ 4: НАСТРОЙКА ПРОЕКТА

## Шаг 7: Создать файл с настройками

### 7.1 Найти файл .env.example:

1. В папке проекта найдите файл `.env.example`
2. Если используете VS Code - он виден в списке файлов слева
3. Если используете проводник (Finder) - включите показ скрытых файлов:
   - **Windows:** View → ✅ Hidden items
   - **Mac:** `Cmd + Shift + .`

### 7.2 Создать файл .env.local:

**Способ 1 - Через VS Code (проще):**
1. Правой кнопкой на файле `.env.example`
2. **Copy** → **Paste**
3. Переименуйте копию в `.env.local`

**Способ 2 - Через терминал:**
```bash
# В папке проекта выполните:
cp .env.example .env.local
```

### 7.3 Заполнить ключи:

1. Откройте файл `.env.local` (двойной клик в VS Code)
2. Увидите:
```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

3. Замените `your_supabase_project_url` на **Project URL** из шага 6
4. Замените `your_supabase_anon_key` на **anon key** из шага 6

**Должно получиться примерно так:**
```env
VITE_SUPABASE_URL=https://abcdefghijklmnop.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoiYW5vbiIsImlhdCI6MTYzMTIwMDAwMCwiZXhwIjoxOTQ2Nzc2MDAwfQ.someLongString
```

5. Сохраните файл (`Ctrl + S` или `Cmd + S`)

✅ **Готово! Настройки сохранены.**

# 🆘 ЧАСТЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

## Проблема 1: "command not found: npm"

**Решение:**
- Node.js не установлен или не в PATH
- Переустановите Node.js
- Перезагрузите компьютер
- Откройте новый терминал

## Проблема 2: Белый экран в браузере

**Решение:**
1. Проверьте консоль браузера (F12)
2. Проверьте что `.env.local` файл создан и заполнен
3. Проверьте что Supabase проект активен
4. Перезапустите сервер (`Ctrl+C`, затем `npm run dev`)

## Проблема 3: "Failed to fetch" ошибки

**Решение:**
1. Проверьте что SQL миграции запущены в Supabase
2. Проверьте правильность ключей в `.env.local`
3. Проверьте что Supabase проект не приостановлен (inactive)

## Проблема 4: Порт 5173 уже занят

**Решение:**
```bash
# Остановите процесс на порту 5173
# Windows:
netstat -ano | findstr :5173
taskkill /PID <номер_процесса> /F

# Mac/Linux:
lsof -ti:5173 | xargs kill -9

# Или используйте другой порт:
npm run dev -- --port 3000
```

## Проблема 5: Медленная работа

**Решение:**
- Это нормально при первом запуске
- Проверьте что не запущено много программ
- Закройте ненужные вкладки браузера
- В production будет быстрее

## 📋 Краткое описание изменений

Расписание для родителей теперь отображает имя ребенка на каждом занятии и показывает только те дни, где есть запланированные занятия (без пустых блоков).

---

## ✨ Новый функционал

### 1. Имя ребенка на каждом занятии

**Что добавлено:**
- ✅ Badge с именем ребенка рядом с названием группы
- ✅ Автоматическое определение ребенка по groupId
- ✅ Визуальное выделение градиентом (изумрудно-золотой)

**Логика:**
```tsx
const getChildNameByGroupId = (groupId: string) => {
  const child = children.find(c => c.groupId === groupId);
  return child ? child.name : null;
};
```

**Пример отображения:**
```
Младшая группа [София]
17:00 - 18:30 | Елена Смирнова | Зал 1
```

### 2. Скрытие пустых дней

**Что изменено:**
- ✅ Показываются ТОЛЬКО дни с запланированными занятиями
- ✅ Добавлен Empty State для недели без занятий
- ✅ Фильтр дней: `weekDays.filter(date => getEventsForDate(date).length > 0)`

**Логика:**
```tsx
const daysWithEvents = weekDays.filter(date => {
  const dayEvents = getEventsForDate(date);
  return dayEvents.length > 0;
});
```

**До изменений:**
```
Понедельник 10 ноября - 2 занятия
Вторник 11 ноября - Занятий нет     ❌ Показывалось
Среда 12 ноября - Занятий нет       ❌ Показывалось  
Четверг 13 ноября - 1 занятие
```

**После изменений:**
```
Понедельник 10 ноября - 2 занятия
Четверг 13 ноября - 1 занятие
```

### 3. Empty State для пустой недели

**Что добавлено:**

{daysWithEvents.length > 0 ? (

## 🔍 Тестовые сценарии

### Сценарий 1: У родителя 2 ребенка в разных группах

**Дано:**
- София в Младшей группе (groupId: '1')
- Даша в Средней группе (groupId: '2')

**Ожидаемый результат:**
```
Понедельник:
  ✅ Младшая группа [София] 16:00-17:30
  ⚪ Старшая группа (нет badge - не ребенок родителя)

Вторник:
  ✅ Средняя группа [Даша] 17:00-18:30
```

### Сценарий 2: Неделя без занятий

**Дано:**
- `events = []` или нет событий на этой неделе

**Ожидаемый результат:**
```
┌────────────────────────────────────┐
│  📅                                │
│  Нет запланированных занятий       │
│  На этой неделе пока нет занятий   │
│  в расписании                      │
└────────────────────────────────────┘
```

### Сценарий 3: Занятие группы, к которой не привязан ребенок

**Дано:**
- Старшая группа (groupId: '3')
- У родителя нет детей в этой группе

**Ожидаемый результат:**
```
Старшая группа       // Без badge
18:00-19:30 | Елена Смирнова | Зал 1
```

### Сценарий 4: Несколько занятий одного ребенка в день

**Дано:**
- София: 2 занятия в понедельник (утро и вечер)

**Ожидаемый результат:**
```
Понедельник:
  ✅ Младшая группа [София] 10:00-11:30
  ✅ Младшая группа [София] 16:00-17:30
```

## 💡 UX улучшения

### До изменений:
```
❌ Проблема 1: Непонятно, какой ребенок идет на занятие
   "Младшая группа 16:00-17:30"
   → А кто из детей? София или Максим?

❌ Проблема 2: Много пустых дней в расписании
   Понедельник: 2 занятия
   Вторник: Занятий нет ⬅️ Занимает место
   Среда: Занятий нет   ⬅️ Занимает место
   Четверг: Занятий нет ⬅️ Занимает место
   Пятница: 1 занятие
   → Много скроллинга, неудобно
```

### После изменений:
```
✅ Решение 1: Имя ребенка сразу видно
   "Младшая группа [София] 16:00-17:30"
   → Сразу понятно, кого вести на занятие!

✅ Решение 2: Компактное расписание
   Понедельник: 2 занятия
   Пятница: 1 занятие
   → Только нужная информация, меньше скроллинга
```

**Преимущества:**
- ✅ Быстрое понимание "кто-когда-куда"
- ✅ Меньше когнитивной нагрузки
- ✅ Компактное отображение
- ✅ Меньше скроллинга
- ✅ Фокус на важном

## 🎯 Связь с данными

### Текущая структура:

```typescript
interface Child {
  id: string;
  name: string;          // ⬅️ Используется в Badge
  groupId: string;       // ⬅️ Связь с Event
  groupName: string;
  // ...
}

interface Event {
  id: string;
  groupId: string;       // ⬅️ Связь с Child
  groupName: string;
  date: Date;
  startTime: string;
  endTime: string;
  // ...
}
```

### Логика связи:

```
Event.groupId === Child.groupId
     ↓
Показываем Child.name в Badge
```

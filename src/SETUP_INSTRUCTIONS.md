# 🚀 CRM "Manera" - Инструкция по настройке Backend

## 📋 Обзор

Ваш CRM полностью готов к работе! Осталось только подключить Supabase.

## 🎯 Быстрый старт (5 минут)

### Шаг 1: Создайте проект в Supabase

1. Перейдите на [supabase.com](https://supabase.com)
2. Нажмите **"Start your project"** или **"New Project"**
3. Заполните:
   - **Name:** `crm-manera`
   - **Database Password:** (сохраните его!)
   - **Region:** Europe (Frankfurt) или ближайший к вам
4. Нажмите **"Create new project"**
5. Дождитесь создания проекта (~2 минуты)

---

### Шаг 2: Запустите SQL миграции

1. В Supabase перейдите в **SQL Editor** (левое меню)
2. Нажмите **"+ New Query"**
3. Скопируйте содержимое файла `/supabase/migrations/001_initial_schema.sql`
4. Вставьте в редактор и нажмите **"Run"** ✅
5. Повторите для файла `/supabase/migrations/002_rls_policies.sql`
6. Повторите для файла `/supabase/migrations/003_seed_data.sql`

**Готово!** База данных создана с тестовыми данными.

---

### Шаг 3: Получите API ключи

1. Перейдите в **Settings** → **API** (левое меню)
2. Найдите:
   - **Project URL** - это ваш `VITE_SUPABASE_URL`
   - **anon public** - это ваш `VITE_SUPABASE_ANON_KEY`

---

### Шаг 4: Настройте проект в Figma Make

1. Скопируйте `.env.example` в `.env.local`
2. Вставьте ваши ключи:
```env
VITE_SUPABASE_URL=https://ваш-проект.supabase.co
VITE_SUPABASE_ANON_KEY=ваш-анон-ключ
```

---

## ✅ Проверка работы

## 🗄️ Структура базы данных

### Основные таблицы:

- **users** - Пользователи системы (4 роли)
- **students** - Ученики
- **parents** - Родители
- **teachers** - Преподаватели
- **groups** - Группы
- **schedules** - Расписание занятий
- **subscriptions** - Абонементы
- **payments** - Платежи
- **attendance** - Посещаемость
- **tasks** - Задачи
- **events** - Мероприятия
- **messages** - Сообщения
- **pricing** - Прайс-лист
- **automations** - Автоматизации
- **analytics_metrics** - Метрики аналитики

### Row Level Security (RLS):

✅ Настроены политики доступа для каждой роли:
- **Родители** видят только своих детей
- **Преподаватели** видят только свои группы
- **Администраторы** видят операционные данные
- **Владелец** видит всё включая финансы

## 🆘 Помощь

### Частые проблемы:

**Ошибка подключения к Supabase:**
- Проверьте правильность URL и ключа
- Убедитесь что проект Supabase активен
- Проверьте наличие файла `.env.local`

**Нет данных:**
- Запустите seed миграцию (`003_seed_data.sql`)
- Проверьте RLS политики
- Проверьте консоль браузера на ошибки

**Ошибки доступа:**
- Проверьте что RLS политики применены
- Убедитесь что пользователь авторизован
- Проверьте роль пользователя в таблице `users`
